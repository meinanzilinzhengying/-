# 第五阶段：探针管理功能

> 前置条件：已完成前四阶段

## 1 阶段目标
实现探针的 SSH 远程安装、资源限制、远程升级、配置下发等管理功能。

## 2 后端开发任务

### 任务 1：SSH 远程安装
- POST /api/v1/probes/install
  - 请求体：{ host, port, username, password, ssh_key, edge_node_id }
  - 实现逻辑：
    1. 后端通过 gRPC 将安装指令发送到指定边缘节点
    2. 边缘节点通过 SSH 连接目标服务器
    3. 检测目标服务器内核版本（>= 4.15）
    4. 上传探针二进制文件到目标服务器 /opt/cloud-flow-probe/
    5. 创建配置文件 /etc/cloud-flow-probe/config.yaml（包含边缘节点地址）
    6. 创建 systemd service 文件
    7. 启动探针服务并设置开机自启
    8. 返回安装结果
  - 支持安装进度推送（通过 WebSocket）

### 任务 2：探针资源限制
- PUT /api/v1/probes/:id/limits
  - 请求体：{ cpu_limit, memory_limit, collection_interval }
  - 实现逻辑：通过 gRPC 下发配置到边缘节点，边缘节点转发给探针
  - 探针收到后更新本地配置并应用资源限制（通过 cgroup 或 Go runtime 调整）

### 任务 3：探针远程升级
- POST /api/v1/probes/upgrade
  - 请求体：{ probe_ids: [], version, package_url }
  - 实现逻辑：
    1. 后端下载新版本探针包到本地
    2. 通过 gRPC 将升级包发送到边缘节点
    3. 边缘节点分发给目标探针
    4. 探针执行滚动升级：停止旧版本 → 替换二进制 → 启动新版本 → 健康检查
    5. 如果升级失败，自动回滚到旧版本
  - 支持批量升级（选择多个探针同时升级）

### 任务 4：探针配置下发
- PUT /api/v1/probes/:id/config
  - 请求体：{ collection_modules: { network: true, tcp: true, http: true, system: true, profiling: false, trace: false }, labels: { env: "production", team: "backend" } }
  - 实现逻辑：通过 gRPC 下发到边缘节点 → 探针

### 任务 5：探针操作 API
- POST /api/v1/probes/:id/restart - 重启探针
- POST /api/v1/probes/:id/stop - 停止探针
- DELETE /api/v1/probes/:id - 卸载探针
- GET /api/v1/probes/:id/logs - 获取探针日志（最近 100 行）
- GET /api/v1/probes/:id/status - 获取探针实时状态（CPU/内存/采集统计）

### 任务 6：边缘节点管理增强
- POST /api/v1/edges - 注册新边缘节点
- PUT /api/v1/edges/:id - 更新边缘节点配置
- DELETE /api/v1/edges/:id - 删除边缘节点
- POST /api/v1/edges/:id/restart - 重启边缘节点
- GET /api/v1/edges/:id/status - 边缘节点状态（CPU/内存/探针数/数据吞吐）

## 3 前端开发任务

### 任务 7：边缘节点管理页
- 边缘节点列表表格（名称、云平台、区域、IP、状态、探针数、数据吞吐）
- 添加边缘节点对话框（名称、云平台、区域、IP、端口）
- 边缘节点详情抽屉（状态指标、探针列表、配置信息）
- 重启/删除操作按钮

### 任务 8：探针管理页
- 探针列表表格（主机名、IP、状态、版本、所属边缘节点、K8s 集群、CPU/内存使用率）
- 筛选栏：边缘节点、K8s 集群、状态、搜索
- SSH 安装对话框：
  - 步骤条（输入信息 → 连接中 → 上传中 → 安装中 → 完成）
  - 输入字段：目标 IP、SSH 端口、用户名、密码/私钥、选择边缘节点
  - 实时进度显示（通过 WebSocket）
  - 安装结果展示（成功/失败+错误信息）
- 探针详情抽屉：
  - 基本信息（IP、版本、内核版本、运行时间）
  - 资源使用（CPU/内存实时图表）
  - 采集统计（各模块采集量）
  - 配置管理（采集模块开关、资源限制设置）
  - 操作按钮（重启、停止、卸载、升级）

### 任务 9：批量操作
- 批量选择探针（表格多选框）
- 批量升级按钮
- 批量配置下发
- 批量重启/停止

### 任务 10：探针日志查看
- 在探针详情抽屉中添加"日志"Tab
- 实时日志流（通过 WebSocket）
- 日志搜索和筛选
