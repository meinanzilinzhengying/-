# 第三阶段：首页仪表盘与网络拓扑

> 前置条件：已完成第一、二阶段（00-全局架构约定.md + 01-基础架构阶段.md + 02-探针核心阶段.md）
> 此时探针已能采集数据并写入 TDengine，边缘节点和探针管理 API 已可用。

## 1 阶段目标
实现首页仪表盘（资源总览 + 资源层级视图 + 实时趋势）和网络拓扑图。

## 2 后端开发任务

### 任务 1：首页概览 API
- GET /api/v1/dashboard/overview
  - 返回：监控节点总数/在线/离线、边缘节点按云平台分布、网络流量概览（实时吞吐/平均延迟/错误率）、告警概览、K8s 资源概览、性能概览
  - 数据来源：DM8（节点/探针/边缘节点统计）+ TDengine（实时流量聚合）+ Redis（缓存 10 秒）

- GET /api/v1/dashboard/trends
  - 参数：time_range（5m/15m/1h/6h/24h）、metric（throughput/latency/error_rate/alert_count）
  - 返回：时序数据点数组 [{timestamp, value}]
  - 数据来源：TDengine 时序查询

- GET /api/v1/dashboard/resources
  - 参数：type（cloud_platform/vpc/cluster/namespace/node）
  - 返回：树形层级数据，每个节点包含 id、name、children、stats（子资源数量和状态）
  - 数据来源：DM8 关联查询

### 任务 2：流量监控 API
- GET /api/v1/traffic/metrics
  - 参数：start_time、end_time、dimension（service/node/pod）、filters
  - 返回：时序数据（吞吐/延迟/重传率/丢包率）
  - 数据来源：TDengine

- GET /api/v1/traffic/top
  - 参数：metric（throughput/latency/error_rate）、top_n（默认 10）
  - 返回：Top N 排行列表
  - 数据来源：TDengine 聚合查询

### 任务 3：拓扑计算引擎
- GET /api/v1/topology/graph
  - 参数：level（vpc/node/pod/service）、namespace、time_range
  - 返回：{ nodes: [{id, name, type, status, metrics}], edges: [{source, target, metrics}] }
  - 实现逻辑：
    1. 从 TDengine 查询时间范围内的流量数据
    2. 按源/目标 IP 聚合，生成节点和边
    3. 注入 K8s 标签（Pod 名、Service 名、Namespace）
    4. 计算异常标记（延迟/错误率超过阈值则标红）

- GET /api/v1/topology/nodes
  - 返回：所有拓扑节点的列表（用于搜索和筛选）

### 任务 4：WebSocket 实时推送
- WebSocket 端点：/api/v1/ws/dashboard
- 客户端连接后，服务端每 1 秒推送：
  - 实时吞吐量、平均延迟、错误率
  - 节点在线/离线状态变化
- 使用 gorilla/websocket，支持 ping/pong 心跳

## 3 前端开发任务

### 任务 5：首页仪表盘组件
**顶部统计卡片（6 个）**：
- 监控节点总数卡片（总数/在线/离线，带趋势箭头）
- 边缘节点状态卡片（按云平台分布的迷你饼图）
- 网络流量概览卡片（实时吞吐/延迟/错误率，带迷你折线图）
- 告警概览卡片（活跃/严重/警告/通知，带颜色区分）
- K8s 资源概览卡片（Pods/Services/Nodes/Deployments 数字）
- 性能概览卡片（CPU/内存使用率环形图）

**中间资源层级视图**：
- 使用 Element Plus 的 el-tree 组件实现树形展开
- 左侧树形结构：云平台 → VPC → K8s 集群 → Namespace → Node
- 右侧详情面板：选中节点的详细信息和关键指标
- 每个层级节点显示子资源数量和健康状态（绿色/红色圆点）

**底部实时趋势图表**：
- 使用 ECharts 绘制 4 个折线图（吞吐、延迟、错误率、告警）
- 时间范围切换按钮组（5分钟/15分钟/1小时/6小时/24小时）
- 通过 WebSocket 实时更新数据

### 任务 6：网络流量监控页
- 顶部筛选栏：命名空间、节点、时间范围选择器
- 主图表区域：ECharts 多轴折线图（吞吐 + 延迟 + 重传率 + 丢包率）
- Top N 排行表格：可切换排序指标
- 协议分布饼图：HTTP/TCP/gRPC 占比

### 任务 7：网络拓扑图
- 使用 AntV G6 绘制交互式拓扑图
- 节点样式：
  - VPC 节点：云朵形状，显示 CIDR
  - Node 节点：矩形，显示 IP 和状态
  - Pod 节点：圆形，显示 Pod 名
  - Service 节点：六边形，显示 Service 名
- 边样式：线条粗细代表流量大小，颜色代表健康状态（绿色正常/红色异常）
- 交互：
  - 节点拖拽、画布缩放
  - 点击节点弹出详情面板（IP、状态、流量指标）
  - 悬浮连线显示流量详情（吞吐/延迟/错误率）
  - 层级切换按钮组（VPC/Node/Pod/Service）
  - Namespace 筛选下拉框

### 任务 8：WebSocket 集成
- 封装 WebSocket 工具类（自动重连、心跳检测）
- 在首页仪表盘中接入 WebSocket 实时数据
- 组件卸载时断开 WebSocket 连接
