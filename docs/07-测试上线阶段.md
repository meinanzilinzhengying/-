# 第七阶段：集成测试与上线部署

> 前置条件：已完成前六阶段，信创适配已完成。

## 1 阶段目标
完成集成测试、性能测试、安全加固、文档完善、正式上线。

## 2 测试任务

### 任务 1：集成测试
编写集成测试用例，覆盖以下场景：

**后端 API 测试**：
- 认证流程：登录 → 获取 Token → 访问受保护接口 → Token 过期 → 重新登录
- 探针管理：注册 → 安装 → 配置 → 升级 → 卸载
- 数据采集：探针启动 → 采集数据 → 边缘汇聚 → 中心存储 → 前端展示
- 告警流程：创建规则 → 触发告警 → 通知发送 → 告警确认 → 告警恢复
- 拓扑计算：数据采集 → 拓扑生成 → 异常检测 → 前端渲染

**前端功能测试**：
- 所有页面正常渲染
- 所有交互功能正常（筛选、分页、搜索、导出）
- WebSocket 实时数据推送正常
- 响应式布局在不同分辨率下正常

**探针功能测试**：
- 网络流量采集准确性
- TCP 指标采集准确性
- 系统指标采集准确性
- K8s 资源感知准确性
- 性能剖析数据准确性
- 断线重连可靠性

### 任务 2：性能测试
- 单探针采集性能：CPU < 2%，内存 < 100MB
- 边缘节点吞吐：支持同时管理 100+ 探针
- 中心服务器查询性能：
  - 首页概览 API 响应时间 < 500ms
  - 拓扑图 API 响应时间 < 2s
  - 时序查询（1 小时范围）< 1s
- 并发测试：100 个并发用户同时访问
- 长时间运行测试：连续运行 72 小时无内存泄漏

### 任务 3：安全加固
- JWT Secret 强制要求修改（安装时随机生成）
- 数据库密码强制要求修改
- API 接口限流（rate limiting）
- SSH 密码加密存储（AES-256）
- HTTPS 支持（Nginx 反向代理配置）
- 操作审计日志完整性检查
- 敏感信息脱敏（日志中不打印密码、Token）

### 任务 4：Nginx 反向代理配置
```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;
    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    # 前端静态文件
    location / {
        root /opt/cloud-flow/web/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # WebSocket 代理
    location /api/v1/ws/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```
