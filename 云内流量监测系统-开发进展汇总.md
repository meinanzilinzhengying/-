# 云内流量监测系统 - 开发进展汇总

> 最后更新：2026-05-03

## 一、项目概述

**项目名称**：云内流量监测系统（eBPF Cloud Flow Platform）

**技术架构**：四层分离

| 模块 | 技术栈 | 职责 |
|------|--------|------|
| **Center** | Go 1.22 + gRPC + TiDB | 中心服务，接收/存储/告警 |
| **Edge** | Go 1.22 + gRPC | 边缘节点，汇聚探针数据并转发 |
| **Agent** | Go 1.22 + eBPF | 探针采集端，采集系统/网络指标 |
| **Frontend** | Vue 3 + Vite + Element Plus + TypeScript | 可视化前端 |
| **Proto** | Go 手写 gRPC 定义 | 共享接口定义（17 个消息 + 2 个服务） |
| **Pkg** | Go 工具包 | 共享工具（grpcutil/ratelimit/trace/utils/storage） |

## 七、部署指南

### 一键启动

```bash
# 解压项目
unzip 云内流量监测系统.zip

# 启动所有服务
docker-compose up -d

# 查看状态
docker-compose ps
```

### 服务端口

| 服务 | 端口 | 说明 |
|------|------|------|
| Frontend (Nginx) | 80 | Web 界面 |
| Center Portal | 8080 | HTTP API |
| Center gRPC | 9090 | gRPC 服务 |
| Edge Agent gRPC | 9091 | 接收探针数据 |
| Edge Center gRPC | 9092 | 转发到 Center |
| Edge Health | 8081 | 健康检查 |
| TiDB | 4000 | MySQL 协议 |
| Redis | 6379 | CSRF/登录失败计数 |
