# 第二阶段：探针核心与边缘节点

> 前置条件：已完成第一阶段（00-全局架构约定.md + 01-基础架构阶段.md）

## 1 阶段目标

开发 eBPF 探针和边缘节点，打通数据采集链路：探针采集 → 边缘节点汇聚 → 中心服务器存储。

## 2 开发任务

### 2.1 Protobuf 协议定义

#### 任务 1：定义 gRPC 协议

在 server/internal/proto/ 下定义：

**probe.proto**（探针 ↔ 边缘节点）：

- ProbeService.Register - 探针注册
- ProbeService.Heartbeat - 心跳
- ProbeService.UploadMetrics - 上报指标（流式）
- ProbeService.ReceiveConfig - 接收配置

**edge.proto**（边缘节点 ↔ 中心服务器）：

- EdgeService.Register - 边缘节点注册
- EdgeService.Heartbeat - 心跳
- EdgeService.UploadMetrics - 上报汇聚指标（流式）
- EdgeService.ReceiveCommand - 接收指令

**common.proto**（公共消息类型）：

- MetricData - 指标数据点
- ProbeInfo - 探针信息
- EdgeInfo - 边缘节点信息
- ConfigUpdate - 配置更新

生成 Go 代码：`protoc --go_out=. --go-grpc_out=. *.proto`

### 2.2 边缘节点开发

#### 任务 2：边缘节点框架

- 创建 edge/cmd/edge/main.go
- gRPC Server（对上连接中心服务器）
- gRPC Server（对下接收探针连接）
- 配置加载（edge/configs/config.yaml）

#### 任务 3：探针连接管理

- 维护探针连接池（map[probeID]grpc.ClientConn）
- 探针注册/注销处理
- 心跳超时检测（30 秒未收到则标记离线）
- 探针配置下发

#### 任务 4：数据汇聚

- 接收多个探针的指标数据
- 数据合并（按时间窗口聚合）
- 标签注入（云平台、VPC、K8s Namespace 等）
- Protobuf 序列化压缩

#### 任务 5：数据转发

- 将汇聚后的数据通过 gRPC 流式发送到中心服务器
- 本地环形缓冲区缓存（最近 5 分钟数据）
- 断线重连 + 数据补发

### 2.3 探针开发

#### 任务 6：探针框架

- 创建 probe/cmd/probe/main.go
- eBPF 程序加载器（加载 .bpf.c 编译后的 .o 文件）
- gRPC 客户端（连接边缘节点）
- 配置加载（probe/configs/config.yaml）

#### 任务 7：网络流量采集（eBPF）

- bpf/network.bpf.c：使用 XDP 在网卡接口挂载 eBPF 程序
  - 采集每个 TCP 连接的字节数、包数
  - 采集 TCP 连接建立/断开事件
  - 使用 tracepoint:sock/inet_sock_set_state
- Go 侧：读取 eBPF perf buffer/map 数据
- 按源IP:端口 → 目标IP:端口 聚合流量

#### 任务 8：TCP 指标采集（eBPF）

- bpf/tcp.bpf.c：
  - tracepoint:tcp/tcp_set_state - 连接状态变更
  - kprobe:tcp_retransmit_skb - 重传事件
  - 统计建连数、断开数、重传数、零窗口次数
- Go 侧：读取并格式化 TCP 指标

#### 任务 9：系统指标采集

- 读取 /proc/stat（CPU 使用率）
- 读取 /proc/meminfo（内存使用）
- 读取 /proc/net/dev（网络 IO）
- 读取 /proc/diskstats（磁盘 IO）
- 每 10 秒采集一次

#### 任务 10：K8s 资源感知

- 监听 K8s apiserver（通过 in-cluster config 或 kubeconfig）
- Watch Pod/Service/Node/Namespace 资源变更
- 维护本地资源缓存
- 将 K8s 标签注入到采集数据中

#### 任务 11：数据上报

- 通过 gRPC 流式上报指标到边缘节点
- 上报间隔：10 秒
- 断线自动重连（指数退避）

### 2.4 中心服务器接收端

#### 任务 12：gRPC 服务端

- 实现 EdgeService（接收边缘节点连接）
- 接收汇聚后的指标数据
- 写入 TDengine

#### 任务 13：边缘节点管理 API

- GET /api/v1/edges - 边缘节点列表
- PUT /api/v1/edges/:id/config - 更新边缘节点配置
- GET /api/v1/edges/:id/probes - 边缘节点下的探针列表

#### 任务 14：探针管理 API（基础）

- GET /api/v1/probes - 探针列表（从 DM8 查询）
- GET /api/v1/probes/:id - 探针详情
- PUT /api/v1/probes/:id/config - 更新探针配置

## 3 验收标准

- [ ] 探针编译成功（需要 Linux 内核 >= 4.15）
- [ ] 探针启动后自动连接边缘节点并注册
- [ ] 探针每 10 秒上报网络流量和系统指标
- [ ] 边缘节点汇聚数据并转发到中心服务器
- [ ] 中心服务器将数据写入 TDengine
- [ ] 前端可查看边缘节点列表和探针列表
- [ ] 探针断线后自动重连
- [ ] 边缘节点本地缓存 5 分钟数据

## 4 TDengine 超级表创建

```sql
CREATE STABLE IF NOT EXISTS flow_metrics (
    ts TIMESTAMP,
    src_ip NCHAR(46),
    src_port INT,
    dst_ip NCHAR(46),
    dst_port INT,
    bytes BIGINT,
    packets BIGINT,
    retransmits BIGINT,
    latency FLOAT
) TAGS (
    probe_id NCHAR(64),
    host_ip NCHAR(46),
    hostname NCHAR(128),
    cloud_platform NCHAR(32),
    region NCHAR(64),
    vpc_id NCHAR(64),
    k8s_cluster NCHAR(64),
    namespace NCHAR(128),
    pod_name NCHAR(256)
);

CREATE STABLE IF NOT EXISTS system_metrics (
    ts TIMESTAMP,
    cpu_usage FLOAT,
    memory_usage FLOAT,
    memory_total BIGINT,
    memory_used BIGINT,
    net_rx_bytes BIGINT,
    net_tx_bytes BIGINT,
    disk_read_bytes BIGINT,
    disk_write_bytes BIGINT
) TAGS (
    probe_id NCHAR(64),
    host_ip NCHAR(46),
    hostname NCHAR(128),
    cpu_count INT
);
```
