# 第六阶段：信创适配与性能优化

> 前置条件：已完成前五阶段，所有核心功能已开发完成。

## 1 阶段目标
适配国产操作系统和国产数据库，进行性能优化，确保在信创环境下稳定运行。

## 2 后端开发任务

### 任务 1：达梦 DM8 适配
- 确认 GORM DM8 驱动兼容性
- 测试所有 SQL 语句在 DM8 上的执行
- 处理 DM8 与 PostgreSQL 的语法差异：
  - 自增序列：DM8 使用 IDENTITY 替代 SERIAL
  - 数据类型映射：TEXT → CLOB、BOOLEAN → INT
  - 分页语法：LIMIT/OFFSET 兼容
  - 字符串函数差异
- 编写 DM8 专用的数据库迁移脚本

### 任务 2：TDengine 国产化验证
- 确认 TDengine 在国产 OS 上的编译和运行
- 测试超级表查询性能
- 配置数据保留策略（30 天自动过期）
- 压缩存储配置优化

### 任务 3：麒麟 V10 适配
- 交叉编译所有 Go 二进制（GOOS=linux GOARCH=amd64/arm64）
- 测试 eBPF 探针在麒麟 V10 内核上的加载：
  - 内核版本检测（麒麟 V10 通常为 4.19 或 5.x）
  - eBPF 程序编译（需要内核头文件）
  - 权限检查（需要 CAP_BPF/CAP_SYS_ADMIN）
- Docker 容器在麒麟上的构建和运行

### 任务 4：统信 UOS 适配
- 同麒麟 V10 的适配流程
- 测试所有组件在 UOS 上的运行

### 任务 5：性能优化
- TDengine 写入优化：批量写入（每 1000 条批量提交一次）
- TDengine 查询优化：合理使用超级表标签过滤
- Redis 缓存优化：首页概览数据缓存 10 秒
- gRPC 流式传输优化：启用 gzip 压缩
- 探针资源优化：CPU 使用控制在 2% 以内，内存控制在 100MB 以内

### 任务 6：安装脚本增强
- 更新 deploy/install.sh 支持国产 OS 检测
- 自动安装 TDengine（国产版本）
- 自动配置达梦 DM8
- 内核版本和 eBPF 支持检测
- 安装前环境检查报告

## 3 前端开发任务

### 任务 7：浏览器兼容性
- 测试在国产浏览器（奇安信安全浏览器、360 安全浏览器）上的兼容性
- 确保不依赖 Chrome-only API
- CSS 兼容性修复

### 任务 8：系统设置页完善
- 数据源配置页：
  - TDengine 连接配置（地址、端口、用户名、密码、数据库名）
  - DM8 连接配置
  - Redis 连接配置
  - 连接测试按钮
- 数据保留策略配置：
  - 保留天数设置（默认 30 天）
  - 各类数据的独立保留策略
- 系统信息页：
  - 版本信息
  - 各组件运行状态
  - 系统资源使用情况（CPU/内存/磁盘）

## 4 验收标准
- [ ] 所有功能在麒麟 V10 上正常运行
- [ ] 所有功能在统信 UOS 上正常运行
- [ ] 达梦 DM8 所有 CRUD 操作正常
- [ ] TDengine 读写性能满足要求
- [ ] eBPF 探针在国产内核上正常加载
- [ ] 安装脚本在国产 OS 上一键安装成功
- [ ] 前端在国产浏览器上正常显示
- [ ] 探针 CPU 使用 < 2%，内存 < 100MB
