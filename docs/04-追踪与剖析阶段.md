# 第四阶段：分布式追踪与性能剖析

> 前置条件：已完成前三阶段

## 1 阶段目标
实现分布式链路追踪、eBPF 性能剖析（CPU/OffCPU/内存）、网络回溯、告警系统。

## 2 后端开发任务

### 任务 1：探针端 - 分布式追踪采集（eBPF）
- bpf/trace.bpf.c：
  - 在 TCP 连接上提取 trace context（trace_id, span_id, parent_span_id）
  - 通过 uprobe hook SSL_read/SSL_write 提取 HTTP 请求头中的 trace 信息
  - 如果没有 trace 信息，自动生成新的 trace_id
  - 记录每个请求的起始时间、结束时间、字节数、状态码
- Go 侧：将 Span 数据通过 gRPC 上报

### 任务 2：探针端 - 性能剖析采集（eBPF）
- bpf/profiling.bpf.c：
  - OnCPU：使用 perf_event PERF_TYPE_HARDWARE 采集 CPU 调用栈
  - OffCPU：使用 tracepoint:sched/sched_switch 采集等待/阻塞调用栈
  - 内存：使用 tracepoint:kmem/mm_page_alloc 采集内存分配调用栈
- Go 侧：
  - 定时触发采集（默认每 60 秒采集 10 秒）
  - 将调用栈符号化（将地址转换为函数名）
  - 通过 gRPC 上报火焰图数据

### 任务 3：追踪 API
- GET /api/v1/traces/list
  - 参数：start_time、end_time、service、status_code、min_duration、max_duration、page、page_size
  - 返回：Trace 列表（trace_id、service、method、path、status_code、duration、timestamp）

- GET /api/v1/traces/:id
  - 返回：完整 Trace 详情（spans 数组，每个 span 包含 span_id、operation_name、start_time、duration、service、tags）

- GET /api/v1/traces/services
  - 返回：所有服务列表及其请求量、错误率、P99 延迟

### 任务 4：性能剖析 API
- GET /api/v1/profiling/cpu
  - 参数：probe_id、host_ip、start_time、end_time
  - 返回：CPU 火焰图数据（折叠调用栈格式，兼容 d3-flame-graph）

- GET /api/v1/profiling/offcpu
  - 参数同上
  - 返回：OffCPU 火焰图数据

- GET /api/v1/profiling/memory
  - 参数同上
  - 返回：内存分配火焰图数据

- GET /api/v1/profiling/funcs
  - 参数：type（cpu/offcpu/memory）、top_n
  - 返回：热点函数排行（函数名、采样次数、占比）

- POST /api/v1/profiling/trace-link
  - 参数：trace_id、time_offset、duration
  - 返回：指定 Trace 时间段内的火焰图数据（Trace-Profile 联动）

### 任务 5：网络回溯 API
- GET /api/v1/forensics/events
  - 参数：start_time、end_time、type（packet_loss/latency_spike/connection_reset）、severity
  - 返回：异常事件列表（时间、类型、影响范围、描述）

- GET /api/v1/forensics/events/:id
  - 返回：事件详情（影响的服务、流量路径、根因分析）

### 任务 6：告警引擎
- 告警规则模型（DM8 alert_rules 表）：
  - metric：监控指标名称
  - condition：条件表达式（gt/lt/gte/lte/eq/neq）
  - threshold：阈值
  - duration：持续时间（秒）
  - severity：严重程度（critical/warning/info）
  - enabled：是否启用

- 告警评估器：
  - 每 10 秒评估一次所有启用的规则
  - 从 TDengine 查询指标值，与阈值比较
  - 触发告警时写入 alert_history 表
  - 支持告警恢复（指标恢复正常时自动 resolve）

- 告警 API：
  - GET /api/v1/alerts/rules - 告警规则列表
  - POST /api/v1/alerts/rules - 创建规则
  - PUT /api/v1/alerts/rules/:id - 更新规则
  - DELETE /api/v1/alerts/rules/:id - 删除规则
  - GET /api/v1/alerts/active - 活跃告警
  - GET /api/v1/alerts/history - 告警历史

### 任务 7：通知渠道
- 支持 Webhook（POST JSON 到指定 URL）
- 支持邮件（SMTP 发送）
- 支持钉钉/企业微信机器人（Webhook）
- 通知渠道配置存储在 DM8

## 3 前端开发任务

### 任务 8：追踪页面
- Trace 列表页：
  - 筛选栏：时间范围、服务名、状态码、最小耗时
  - 表格列：Trace ID、服务、方法、路径、状态码、耗时、时间
  - 点击行跳转详情
- Trace 详情页：
  - 瀑布图（每个 Span 一行，显示操作名、耗时、服务名）
  - Span 颜色编码：绿色正常/黄色慢/红色错误
  - 点击 Span 展开详情（标签、日志）
  - 服务调用 DAG 图（使用 G6 绘制）

### 任务 9：性能剖析页面
- Tab 切换：OnCPU / OffCPU / 内存
- 进程列表（左侧）：显示所有探针采集的进程，点击切换
- 火焰图（主区域）：使用 d3-flame-graph 组件渲染
  - 悬浮显示函数名、采样次数、占比
  - 点击函数可下钻
- 热点函数排行（右侧）：表格显示 Top 10 热点函数
- Trace 联动按钮：输入 Trace ID，查看对应时间段的火焰图

### 任务 10：网络回溯页面
- 异常事件时间线（垂直时间轴组件）
- 每个事件卡片显示：时间、类型图标、描述、影响范围
- 点击事件展开详情面板
- 事件筛选：类型、严重程度、时间范围

### 任务 11：告警管理页面
- 告警规则页：
  - 规则列表表格
  - 创建/编辑规则的表单对话框（指标、条件、阈值、持续时间、严重程度）
  - 启用/禁用开关
- 活跃告警页：
  - 告警列表（按严重程度排序，红色/黄色/蓝色标识）
  - 确认/静默操作按钮
- 告警历史页：
  - 历史告警表格
  - 筛选：时间范围、严重程度、状态
- 通知渠道配置页：
  - Webhook/邮件/钉钉/企业微信的配置表单

## 4 TDengine 新增表
```sql
CREATE STABLE IF NOT EXISTS trace_spans (
    ts TIMESTAMP,
    trace_id NCHAR(64),
    span_id NCHAR(32),
    parent_span_id NCHAR(32),
    operation_name NCHAR(256),
    service_name NCHAR(128),
    start_time BIGINT,
    duration BIGINT,
    status_code INT,
    tags JSON,
    http_method NCHAR(16),
    http_url NCHAR(512),
    http_status_code INT
) TAGS (
    probe_id NCHAR(64),
    host_ip NCHAR(46)
);

CREATE STABLE IF NOT EXISTS profiling_data (
    ts TIMESTAMP,
    profile_type NCHAR(16),
    stack_data NCHAR(65535),
    total_samples BIGINT,
    duration_ns BIGINT
) TAGS (
    probe_id NCHAR(64),
    host_ip NCHAR(46),
    pid INT,
    comm NCHAR(128)
);
```
